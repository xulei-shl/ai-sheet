# AI-Sheet 核心页面 UI/UX 深度评估与重构建议报告

本报告针对`公式处理`、`LLM处理`、`配置管理`、`提示词管理` 4 个核心功能页面的设计与交互进行了评估，并结合新版「数据加载」的**「左右分栏工作空间」**设计语言，提出了具体的优化思路。

---

## 1. 公式处理页面 (FormulaPage.tsx)

### 🔴 当前痛点分析
- **原生下拉选择器**：文件和 Sheet 依然使用原生 `<select>` 下拉框，在大屏幕上显得廉价且交互低效，且与「数据加载」侧边树的结构割裂。
- **目标列选择杂乱**：当选择“覆盖现有列”时，列名以下拉框展开。如果 Excel 有数十个列，下拉查找极其繁琐。
- **垂直空间积压严重**：所有选项（文件、Sheet、策略、列、公式 textarea、操作按钮）塞在一个大卡片中，将最核心的“预览表格”推到了屏幕极下方，导致用户必须频繁滚动。
- **历史记录隐蔽**：公式历史记录折叠在大面板底部，不易发现，加载历史的交互不够连观。

### 💡 优化与重构设计方案
- **共享/同步左侧树**：
  - 引入与数据加载页一致的左侧文件/Sheet 树（或直接复用 store 里的聚焦状态），点击 Sheet 直接切换右侧公式工作区。
- **工作区 Side-by-Side 布局**：
  - **左半侧（配置栏）**：保留公式输入 `textarea`，但目标列选择可以采用像 `ColumnSelector` 一样的网格选择器，仅允许单选，并支持搜索。
  - **右半侧（历史与模板库）**：将公式历史记录常驻，并提供一些“常用 Excel 公式模板”（如 `CONCAT`, `UPPER`, `IF`），点击一键填入编辑器。
- **下半侧（预览表格）**：
  - 常驻显示数据预览表格，公式编写时，表格中的目标列实时展示公式计算后的预览效果，提供沉浸式反馈。

## 2. LLM 处理页面 (AiPage.tsx / LLMProcessingPage.tsx / PythonProcessingPage.tsx)

### 🔴 当前痛点分析
- **输入列药丸按钮墙**：在 `LLMProcessingPage` 中，输入列（`inputColumns`）同样非常多，当前直接以 `flex-wrap` 平铺，导致列多时瞬间铺满屏幕，无搜索过滤，极度杂乱。
- **配置与日志上下挤压**：左侧/上方是大量设置（参数、提示词、温度），下方是处理进度条和处理日志。一旦开始处理，用户需要同时盯着进度和日志，当前的上下堆叠布局导致视线上下漂移，且配置项过多时会完全遮挡日志。
- **Python 编辑器狭窄**：`PythonProcessingPage` 的脚本输入框是一块高度固定的 `textarea`，执行输出（终端）在最下方，编辑和调试体验较差，不像一个专业的代码工作区。

### 💡 优化与重构设计方案
- **输入列选择器升级**：
  - 直接引入重构后的 `ColumnSelector`（支持搜索过滤与紧凑网格），收纳在最大高度 `max-h-48` 内。
- **「配置 | 执行监控」双栏布局**：
  - 将页面重构为双栏：
    - **左侧配置面板**：包含文件/Sheet信息（可折叠）、输入列/输出列选择、提示词（下拉/自定义文本域）、模型参数。
    - **右侧执行控制台（Console）**：上方为大卡片进度条（包含速度、预计时间、环形图进度），下方为占满剩余空间的日志终端（使用类似命令行的暗色背景，高亮展示 Info/Warning/Error 等级别，支持一键清空和日志搜索）。
- **Python 处理页 — Notebook 分栏模式**：
  - 采用 **左代码、右终端** 布局。左侧使用全高代码输入区，右侧为带标题栏的模拟终端输出区，完美模拟 Jupyter Notebook 体验。

## 3. 配置管理页面 (ConfigPage.tsx)

### 🔴 当前痛点分析
- **表单打断列表流**：新建和编辑模型时，表单卡片是在列表上方或中间直接展开的，当模型数量较多时，展开的表单会把其他模型卡片挤出屏幕，产生强烈的跳跃感。
- **信息展示松散**：模型卡片设计偏大，信息利用率低，多模型自动降级的先后顺序（优先级）不够直观。

### 💡 优化与重构设计方案
- **「列表-详情（Master-Detail）」侧并侧布局**：
  - **左侧栏（模型列表）**：
    - 展示所有的模型项，以紧凑列表形式呈现。显示模型名称、是否默认状态。
    - 支持通过拖拽（Drag and Drop）调整列表顺序，直观地决定多模型自动降级（Fallback）的先后顺序。
  - **右侧栏（配置详情面板）**：
    - 点击左侧任意模型或“新增”按钮，右侧面板即时显示或创建其对应的配置表单（含 API 密钥、URL、Provider 选择，以及「测试连接」按钮）。
    - 这样设计可以确保列表部分结构稳定，表单拥有极宽敞和规整的展示空间。

## 4. 提示词管理页面 (PromptsPage.tsx)

### 🔴 当前痛点分析
- **文本预览憋屈**：提示词内容通常是一大段 Markdown 文本，目前塞在卡片里仅限制了 `max-h-24` 预览，想看全部内容必须点击编辑。
- **无变量感知**：提示词通常包含 `{{variable}}` 占位符变量，当前没有针对这些占位符进行高亮或列出的视觉交互。

### 💡 优化与重构设计方案
- **「库导航-编辑器」双侧分栏布局**：
  - **左侧栏（提示词检索）**：
    - 顶部为搜索框，下方为分类筛选标签页（如：全部 / 翻译 / 摘要 / 代码）。
    - 列表展示提示词名称、最后修改时间和简短摘要。
  - **右侧栏（大文本编辑器）**：
    - 显示完整的 Prompt 编辑面板。
    - 文本输入框使用 monospace 字体，并在右侧或下方提供一个“提取的变量（Detected Variables）”提示区，自动解析出 `{{A}}` 等变量，方便用户校对。
    - 提供“测试运行”（跳转至 LLM 处理）和“一键复制”等快捷操作。

## 总结：全局一致的设计语言（Shared Visual DNA）

若将以上四个页面全部按照分栏和左右工作空间的模式进行升级，整个 AI-Sheet 应用将形成高度统一的视觉 DNA：

| 页面 | 左侧栏 (Left Sidebar) | 主工作区 (Main Workspace) / 右侧栏 |
|---|---|---|
| **数据加载** | 文件与 Sheet 导航树 (Context 勾选) | 激活 Sheet 列网格选择器 + 数据预览表格 |
| **公式处理** | 共享文件树 + 快捷公式模板库 | 公式编写与目标列配置 + 实时预览表格 |
| **LLM处理** | LLM 配置项 (输入/输出列网格、提示词与温度) | 处理进度面板 + 全高命令行日志终端 |
| **配置管理** | 模型列表 (支持降级顺序拖拽调整) | 激活模型的详细 API Form + 连接测试面板 |
| **提示词管理**| 提示词按分类检索列表 | Monospace 编辑器 + 占位符变量检测面板 |
