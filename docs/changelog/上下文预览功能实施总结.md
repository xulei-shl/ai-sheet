# 上下文预览功能 - 实施总结

## 概述

成功实现了 Agent 右侧面板的上下文预览功能，用户现在可以清晰地看到 AI Agent 已加载的文件、列和数据样本。

## 实现的功能

### 1. 核心功能
- ✅ 实时显示已加载的 Excel 文件列表
- ✅ 显示用户选择的数据列
- ✅ 展示数据预览（前 3 行）
- ✅ 可折叠/展开的卡片设计
- ✅ 与数据操作实时同步

### 2. 技术实现

#### 类型系统扩展
```typescript
// src/types/agent.ts
interface LoadedFile {
  name: string;
  path: string;
  sheet?: string;

interface SampleDataPreview {

  rows: string[][];
```

- **agentStore**: 新增 `loadedContext` 状态和 `setLoadedContext()` 方法
- **excelStore**: 在所有数据变更点调用 `notifyContextChange()`，同时更新前端和后端上下文

## 用户使用流程

1. 用户在"数据管理"页面上传 Excel 文件
2. 选择 Sheet 和列
3. 切换到右侧 Agent 面板
4. 看到上下文预览卡片自动显示：

   - 选中的列

5. 点击展开/折叠按钮查看详情
6. 与 Agent 对话时，Agent 已感知这些上下文

## 设计考量

### 为什么分离前端和后端上下文？

**前端上下文 (richContext)**:
- 包含完整的 UI 展示所需数据
- 包括 `LoadedFile` 对象（name, path, sheet）
- 包括 `SampleDataPreview`（columns, rows）

**后端上下文 (backendContext)**:
- 仅包含 Agent 推理所需的关键信息
- 简化的文件引用（避免传输大量数据）
- 与 Rust/Node.js 协议兼容

### 性能优化

1. **按需更新**: 仅在数据变更时触发 `notifyContextChange()`
2. **数据限制**: 预览仅显示前 3 行（可配置）
3. **懒加载**: 折叠状态默认关闭，减少初始渲染开销

### 可访问性

- ✅ `aria-expanded` 标记折叠状态
- ✅ `aria-label` 标记图标按钮
- ✅ 键盘可访问（Enter/Space 展开折叠）
- ✅ 高对比度文本（符合 WCAG AA）
