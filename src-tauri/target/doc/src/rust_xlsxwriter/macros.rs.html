3# Working with VBA Macros
4
5
6This section explains how to add a VBA file containing functions or macros to an
7`rust_xlsxwriter` file.
8
9<img src="https://rustxlsxwriter.github.io/images/app_macros.png">
10
11# The Excel XLSM file format
12
13An Excel `xlsm` file is structurally the same as an `xlsx` file except that it
14contains an additional `vbaProject.bin` binary file containing VBA functions
15and/or macros.
16
17Excel `xlsm` files are subject to additional security checks and warning when
18loaded:
19
20<img src="https://rustxlsxwriter.github.io/images/doc_macros_warning.png">
21
22
23# How VBA macros are included in `rust_xlsxwriter`
24
25The `vbaProject.bin` in a `xlsm` file is a binary OLE COM container. This was
26the format used in older `xls` versions of Excel prior to Excel 2007. Unlike
27other components of an xlsx/xlsm file the data isn't stored in XML format.
28Instead the functions and macros as stored as a pre-parsed binary format. As
29such it wouldn't be feasible to programmatically define macros and create a
30`vbaProject.bin` file from scratch (at least not in the remaining lifespan and
31interest levels of the author).
32
33Instead, as a workaround, a utility is used to extract `vbaProject.bin` files
34from existing xlsm files which you can then add to `rust_xlsxwriter` files.
35
36
37# The `vba_extract` utility
38
39The Rust [`vba_extract`](https://crates.io/crates/vba_extract) utility is used
40to extract the `vbaProject.bin` binary from an Excel `xlsm` file. The utility
41can be installed via `cargo`:
42
43```bash
44$ cargo install vba_extract
45```
46
47Once `vba_extract` is installed it can be used as follows:
48
49```bash
50$ vba_extract macro_file.xlsm
51
52Extracted: vbaProject.bin
53```
54
55If the VBA project is signed, `vba_extract` also extracts the
56`vbaProjectSignature.bin` file from the xlsm file (see below).
57
58The syntax and options for `vba_extract` are:
59
60```text
61$ vba_extract --help
62
63Utility to extract a `vbaProject.bin` binary from an Excel xlsm macro file
64for insertion into an `rust_xlsxwriter` file. If the macros are digitally
65signed, it also extracts a `vbaProjectSignature.bin` file.
66
67Usage: vba_extract [OPTIONS] <FILENAME_XLSM>
68
69Arguments:
70  <FILENAME_XLSM>
71          Input Excel xlsm filename
72
73Options:
74  -o, --output-macro-filename <OUTPUT_MACRO_FILENAME>
75          Output vba macro filename
76
77          [default: vbaProject.bin]
78
79  -s, --output-sig-filename <OUTPUT_SIG_FILENAME>
80          Output vba signature filename (if present in the parent file)
81
82          [default: vbaProjectSignature.bin]
83
84  -h, --help
85          Print help (see a summary with '-h')
86
87  -V, --version
88          Print version
89```

92# Adding VBA macros to a `rust_xlsxwriter` file
93
94Once the `vbaProject.bin` file has been extracted it can be added to the
95`rust_xlsxwriter` workbook using the
96[`Workbook::add_vba_project()`](crate::Workbook::add_vba_project()) method:
97
98```
99# // This code is available in examples/doc_macros_add.rs
100# use rust_xlsxwriter::{Workbook, XlsxError};
101#
102# #[allow(unused_variables)]
103# fn main() -> Result<(), XlsxError> {
104#     let mut workbook = Workbook::new();
105#
106    workbook.add_vba_project("examples/vbaProject.bin")?;
107#
108#     Ok(())
109# }
110```
111
112
113Here is a complete example which adds a macro file with a dialog. It also uses a
114button, via [`Worksheet::insert_button()`](crate::Worksheet::insert_button), to
115trigger the macro:
116
117```
118# // This code is available in examples/app_macros.rs
119use rust_xlsxwriter::{Button, Workbook, XlsxError};
120
121fn main() -> Result<(), XlsxError> {
122    // Create a new Excel file object.
123    let mut workbook = Workbook::new();
124
125    // Add the VBA macro file.
126    workbook.add_vba_project("examples/vbaProject.bin")?;
127
128    // Add a worksheet and some text.
129    let worksheet = workbook.add_worksheet();
130
131    // Widen the first column for clarity.
132    worksheet.set_column_width(0, 30)?;
133
134    worksheet.write(2, 0, "Press the button to say hello:")?;
135
136    // Add a button tied to a macro in the VBA project.
137    let button = Button::new()
138        .set_caption("Press Me")
139        .set_macro("say_hello")
140        .set_width(80)
141        .set_height(30);
142
143    worksheet.insert_button(2, 1, &button)?;
144
145    // Save the file to disk. Note the `.xlsm` extension. This is required by
146    // Excel or it raise a warning.
147    workbook.save("macros.xlsm")?;
148
149    Ok(())
150}
151```
152The macro in this example is the following VBA code:
153
154```basic
155Sub say_hello()
156    MsgBox ("Hello from Rust!")
157End Sub
158```

215# Setting the VBA object names
216
217VBA macros generally refer to workbook and worksheet objects via names such as
218`ThisWorkbook` and `Sheet1`, `Sheet2` etc.
219
220If the imported macro uses other names you can set them using the
221[`Workbook::set_vba_name()`](crate::Workbook::set_vba_name()) and
222[`Worksheet::set_vba_name()`](crate::Worksheet::set_vba_name()) methods as
223follows.
224
225```
226# // This code is available in examples/doc_macros_name.rs
227# use rust_xlsxwriter::{Workbook, XlsxError};
228#
229# fn main() -> Result<(), XlsxError> {
230#     let mut workbook = Workbook::new();
231#
232#     workbook.add_vba_project("examples/vbaProject.bin")?;
233    workbook.set_vba_name("MyWorkbook")?;
234#
235#     let worksheet = workbook.add_worksheet();
236    worksheet.set_vba_name("MySheet1")?;
237#
238#     // Note the `.xlsm` extension.
239#     workbook.save("macros.xlsm")?;
240#
241#     Ok(())
242# }
243```
244
245**Note**: If you are using a non-English version of Excel you need to pay
246particular attention to the workbook/worksheet naming that your version of Excel
247uses and add the correct VBA names. You can find the names that are used in the
248VBA editor:
249
250<img src="https://rustxlsxwriter.github.io/images/doc_macros_editor.png">
251
252You can also find them by unzipping the `xlsm` file and grepping the component
253XML files. The following shows how to do that using system `unzip` and libxml's
254[xmllint](http://xmlsoft.org/xmllint.html) to format the XML for clarity
255
256```bash
257$ unzip myfile.xlsm -d myfile
258$ xmllint --format `find myfile -name "*.xml" | xargs` | grep "Pr.*codeName"
259
260    <workbookPr codeName="MyWorkbook" defaultThemeVersion="124226"/>
261    <sheetPr codeName="MySheet"/>
262```

264# Adding a VBA macro signature file to an `rust_xlsxwriter` file
265
266VBA macros can be signed in Excel to allow for further control over execution.
267The signature part is added to the `xlsm` file in another binary called `vbaProjectSignature.bin`.
268
269The `vba_extract` utility will extract the `vbaProject.bin` and
270`vbaProjectSignature.bin` files from an `xlsm` file with signed macros.
271
272These files can be added to a `rust_xlsxwriter` file using the
273[`Workbook::add_vba_project_with_signature()`](crate::Workbook::add_vba_project_with_signature())
274method:
275
276```
277# // This code is available in examples/doc_macros_signed.rs
278# use rust_xlsxwriter::{Workbook, XlsxError};
279#
280# #[allow(unused_variables)]
281# fn main() -> Result<(), XlsxError> {
282#     let mut workbook = Workbook::new();
283#
284    workbook.add_vba_project_with_signature(
285        "examples/vbaProject.bin",
286        "examples/vbaProjectSignature.bin",
287    )?;
288#
289#     let worksheet = workbook.add_worksheet();
290#
291#     // Note the `.xlsm` extension.
292#     workbook.save("macros.xlsm")?;
293#
294#     Ok(())
295# }
296```

298# What to do if it doesn't work
299
300The `rust_xlsxwriter` test suite contains several tests to ensure that this
301feature works and there is a working example shown above. However, there is no
302guarantee that it will work in all cases. Some trial and error may be required
303and some knowledge of VBA will certainly help. If things don't work out here are
304some things to try:
305
3061. Start with a simple macro file, ensure that it works, and then add
307   complexity.
308
3092. Check the code names that macros use to refer to the workbook and worksheets
310   (see above). In general VBA uses a code name of
311   `ThisWorkbook` to refer to the current workbook and the sheet name (such as
312   `Sheet1`) to refer to the worksheets. These are the defaults used by
313   `rust_xlsxwriter`. If the macro uses other names, or the macro was extracted
314   from an non-English language version of Excel, then you can specify these
315   using the workbook and worksheet `set_vba_name` methods.
