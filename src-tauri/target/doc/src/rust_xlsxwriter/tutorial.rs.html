21# Introduction
22
23The `rust_xlsxwriter` library is a Rust library for writing Excel files in
24the xlsx format.
25
26<img src="https://rustxlsxwriter.github.io/images/demo.png">
27
28It can be used to write text, numbers, dates and formulas to multiple worksheets
29in a new Excel 2007+ xlsx file. It has a focus on performance and on fidelity
30with the file format created by Excel.
31
32This document is a tutorial on getting started with `rust_xlsxwriter` and for
33using it to write Excel xlsx files.
34
35# Getting started
36
37To use the `rust_xlsxwriter` in an application or in another library you
38will need add it as a dependency to the `Cargo.toml` file of your project.
39
40To demonstrate the steps required we will start with a small sample application.
41
42## Create a sample project
43
44Create a new Rust command-line application as follows:
45
46```bash
47$ cargo new hello-xlsx
48```
49
50Change to the new `hello-xlsx` directory:
51
52```bash
53$ cd hello-xlsx
54```
55
56The directory structure will look like the following:
57
58```bash
59hello-xlsx/
60├── Cargo.toml
61└── src
62    └── main.rs
63```
64
65## Add `rust_xlsxwriter` to Cargo.toml
66
67Add the `rust_xlsxwriter` dependency to the project `Cargo.toml` file:
68
69```bash
70$ cargo add rust_xlsxwriter
71```
72
73## Modify main.rs
74
75Modify the `src/main.rs` file so it looks like this:
76
77```rust
78// This code is available in examples/app_hello_world.rs
79use rust_xlsxwriter::{Workbook, XlsxError};
80
81fn main() -> Result<(), XlsxError> {
82    // Create a new Excel file object.
83    let mut workbook = Workbook::new();
84
85    // Add a worksheet to the workbook.
86    let worksheet = workbook.add_worksheet();
87
88    // Write a string to cell (0, 0) = A1.
89    worksheet.write(0, 0, "Hello")?;
90
91    // Write a number to cell (1, 0) = A2.
92    worksheet.write(1, 0, 12345)?;
93
94    // Save the file to disk.
95    workbook.save("hello.xlsx")?;
96
97    Ok(())
98}
99```
100
101## Run the application
102
103Run the application as follows:
104
105```bash
106$ cargo run
107```
108
109This will create an output file called `hello.xlsx` which should look something
110like this:
111
112<img src="https://rustxlsxwriter.github.io/images/hello.png">
113
114Once you have the "hello world" application working you can move on to a
115slightly more realistic tutorial example.

134## Tutorial Part 1: Adding data to a worksheet
135
136To add some sample expense data to a worksheet we could start with a simple
137program like the following:
138
139```rust
140// This code is available in examples/app_tutorial1.rs
141use rust_xlsxwriter::{Formula, Workbook, XlsxError};
142
143fn main() -> Result<(), XlsxError> {
144    // Some sample data we want to write to a spreadsheet.
145    let expenses = vec![
146        ("Rent", 2000),
147        ("Gas", 200),
148        ("Food", 500),
149        ("Gym", 100),
150    ];
151
152    // Create a new Excel file object.
153    let mut workbook = Workbook::new();
154
155    // Add a worksheet to the workbook.
156    let worksheet = workbook.add_worksheet();
157
158    // Iterate over the data and write it out row by row.
159    let mut row = 0;
160    for expense in &expenses {
161        worksheet.write(row, 0, expense.0)?;
162        worksheet.write(row, 1, expense.1)?;
163        row += 1;
164    }
165
166    // Write a total using a formula.
167    worksheet.write(row, 0, "Total")?;
168    worksheet.write(row, 1, Formula::new("=SUM(B1:B4)"))?;
169
170    // Save the file to disk.
171    workbook.save("tutorial1.xlsx")?;
172
173    Ok(())
174}
175```
176
177If we run this program we should get a spreadsheet that looks like this:
178
179<img src="https://rustxlsxwriter.github.io/images/tutorial1.png">
180
181This is a simple program but it demonstrates some of the steps that would apply
182to any `rust_xlsxwriter` program.
183
184
185The first step is to create a new workbook object using the
186[`Workbook`](crate::Workbook) constructor
187[`Workbook::new()`](crate::Workbook::new):
188
189
190```text
191let mut workbook = Workbook::new();
192```
193
194**Note**, `rust_xlsxwriter` can only create new files. It cannot read or modify
195existing files.
196
197The workbook object is then used to add a new worksheet via the
198[`Workbook::add_worksheet()`](crate::Workbook::add_worksheet) method:
199
200```text
201let worksheet = workbook.add_worksheet();
202```
203The worksheet will have a standard Excel name, in this case "Sheet1". You can
204specify the worksheet name using the
205[`Worksheet::set_name()`](crate::Worksheet::set_name) method.
206
207
208We then iterate over the data and use the
209[`Worksheet::write()`](crate::Worksheet::write) method which converts common Rust
210types to the equivalent Excel types and writes them to the specified `row, col`
211location in the worksheet:
212
213```text
214    for expense in &expenses {
215        worksheet.write(row, 0, expense.0)?;
216        worksheet.write(row, 1, expense.1)?;
217        row += 1;
218    }
219```
220
221> **Reading ahead**:
222>
223> There are other type specific write methods such as
224> [`Worksheet::write_string()`](crate::Worksheet::write_string) and
225> [`Worksheet::write_number()`](crate::Worksheet::write_number). However, these
226> aren't generally required and thanks to Rust's monomorphization the
227> performance of the generic `write()` method is just as fast.
228>
229> There are also worksheet methods for writing arrays of data or arrays of
230> arrays of data that can be useful in cases where the data to be added is in
231> a vector format:
232>
233> - [`Worksheet::write_row()`](crate::Worksheet::write_row)
234> - [`Worksheet::write_column()`](crate::Worksheet::write_column)
235> - [`Worksheet::write_row_matrix()`](crate::Worksheet::write_row_matrix)
236> - [`Worksheet::write_column_matrix()`](crate::Worksheet::write_column_matrix)
237> - [`Worksheet::write_row_with_format()`](crate::Worksheet::write_row_with_format)
238> - [`Worksheet::write_column_with_format()`](crate::Worksheet::write_column_with_format)
239
240Throughout `rust_xlsxwriter` rows and columns are zero indexed. So the first
241cell in a worksheet `(0, 0)` is equivalent to the Excel notation of `A1`.
242
243To calculate the total of the items in the second column we add a
244[`Formula`](crate::Formula):
245
246```text
247    worksheet.write(row, 1, Formula::new("=SUM(B1:B4)"))?;
248```
249
250Finally, we save and close the Excel file via the
251[`Workbook::save()`](crate::Workbook::save) method which will generate the
252spreadsheet shown in the image above.:
253
254```text
255    workbook.save("tutorial1.xlsx")?;
256```
257
258> **Reading ahead**:
259>
260> The [`Workbook::save()`](crate::Workbook::save) method takes a [`std::path`]
261> argument which can be a `Path`, `PathBuf` or a filename string. It is also
262> possible to save to a byte vector using
263> [`Workbook::save_to_buffer()`](crate::Workbook::save_to_buffer).

266## Tutorial Part 2: Adding some formatting
267
268The previous example converted the required data into an Excel file but it
269looked a little bare. To make the information clearer we can add some
270simple formatting, like this:
271
272<img src="https://rustxlsxwriter.github.io/images/tutorial2.png">
273
274
275The differences here are that we have added "Item" and "Cost" column headers in
276a bold font, we have formatted the currency in the second column and we have
277made the "Total" string bold.
278
279To do this programmatically we can extend our code as follows:
280
281
282```rust
283// This code is available in examples/app_tutorial2.rs
284use rust_xlsxwriter::{Format, Formula, Workbook, XlsxError};
285
286fn main() -> Result<(), XlsxError> {
287    // Some sample data we want to write to a spreadsheet.
288    let expenses = vec![
289        ("Rent", 2000),
290        ("Gas", 200),
291        ("Food", 500),
292        ("Gym", 100),
293    ];
294
295    // Create a new Excel file object.
296    let mut workbook = Workbook::new();
297
298    // Add a bold format to use to highlight cells.
299    let bold = Format::new().set_bold();
300
301    // Add a number format for cells with money values.
302    let money_format = Format::new().set_num_format("$#,##0");
303
304    // Add a worksheet to the workbook.
305    let worksheet = workbook.add_worksheet();
306
307    // Write some column headers.
308    worksheet.write_with_format(0, 0, "Item", &bold)?;
309    worksheet.write_with_format(0, 1, "Cost", &bold)?;
310
311    // Iterate over the data and write it out row by row.
312    let mut row = 1;
313    for expense in &expenses {
314        worksheet.write(row, 0, expense.0)?;
315        worksheet.write_with_format(row, 1, expense.1, &money_format)?;
316        row += 1;
317    }
318
319    // Write a total using a formula.
320    worksheet.write_with_format(row, 0, "Total", &bold)?;
321    worksheet.write_with_format(row, 1, Formula::new("=SUM(B2:B5)"), &money_format)?;
322
323    // Save the file to disk.
324    workbook.save("tutorial2.xlsx")?;
325
326    Ok(())
327}
328```
329
330The main difference between this and the previous program is that we have added
331two [`Format`](crate::Format) objects that we can use to format cells in the
332spreadsheet.
333
334`Format` objects represent all the formatting properties that can be applied
335to a cell in Excel such as fonts, number formatting, colors and borders. This is
336explained in more detail in the [`Format`](crate::Format) struct documentation.
337
338For now we will avoid getting into the details of `Format` and just use a
339limited amount of the its functionality to add some simple formatting:
340
341```text
342    // Add a bold format to use to highlight cells.
343    let bold = Format::new().set_bold();
344
345    // Add a number format for cells with money values.
346    let money_format = Format::new().set_num_format("$#,##0");
347```
348
349We can use these formats with the
350[`Worksheet::write_with_format()`](crate::Worksheet::write_with_format) method
351which writes data and formatting together, like these examples from the code:
352
353```text
354    worksheet.write_with_format(0, 0, "Item", &bold)?;
355
356    worksheet.write_with_format(row, 1, expense.1, &money_format)?;
357
358    worksheet.write_with_format(row, 1, Formula::new("=SUM(B2:B5)"), &money_format)?;
359```

361## Tutorial Part 3: Adding dates and more formatting
362
363Let's extend the application a little bit more to add some dates to the data:
364
365```rust
366    let expenses = [
367        ("Rent", 2000, "2022-09-01"),
368        ("Gas", 200, "2022-09-05"),
369        ("Food", 500, "2022-09-21"),
370        ("Gym", 100, "2022-09-28"),
371    ];
372```
373
374The corresponding spreadsheet will look like this:
375
376<img src="https://rustxlsxwriter.github.io/images/tutorial3.png">
377
378The differences here are that we have added a "Date" column with formatting and
379made that column a little wider to accommodate the dates.
380
381To do this we can extend our program as follows:
382
383```rust
384// This code is available in examples/app_tutorial3.rs
385use rust_xlsxwriter::{ExcelDateTime, Format, Formula, Workbook, XlsxError};
386
387fn main() -> Result<(), XlsxError> {
388    // Some sample data we want to write to a spreadsheet.
389    let expenses = vec![
390        ("Rent", 2000, "2022-09-01"),
391        ("Gas", 200, "2022-09-05"),
392        ("Food", 500, "2022-09-21"),
393        ("Gym", 100, "2022-09-28"),
394    ];
395
396    // Create a new Excel file object.
397    let mut workbook = Workbook::new();
398
399    // Add a bold format to use to highlight cells.
400    let bold = Format::new().set_bold();
401
402    // Add a number format for cells with money values.
403    let money_format = Format::new().set_num_format("$#,##0");
404
405    // Add a number format for cells with dates.
406    let date_format = Format::new().set_num_format("d mmm yyyy");
407
408    // Add a worksheet to the workbook.
409    let worksheet = workbook.add_worksheet();
410
411    // Write some column headers.
412    worksheet.write_with_format(0, 0, "Item", &bold)?;
413    worksheet.write_with_format(0, 1, "Cost", &bold)?;
414    worksheet.write_with_format(0, 2, "Date", &bold)?;
415
416    // Adjust the date column width for clarity.
417    worksheet.set_column_width(2, 15)?;
418
419    // Iterate over the data and write it out row by row.
420    let mut row = 1;
421    for expense in &expenses {
422        worksheet.write(row, 0, expense.0)?;
423        worksheet.write_with_format(row, 1, expense.1, &money_format)?;
424
425        let date = ExcelDateTime::parse_from_str(expense.2)?;
426        worksheet.write_with_format(row, 2, &date, &date_format)?;
427
428        row += 1;
429    }
430
431    // Write a total using a formula.
432    worksheet.write_with_format(row, 0, "Total", &bold)?;
433    worksheet.write_with_format(row, 1, Formula::new("=SUM(B2:B5)"), &money_format)?;
434
435    // Save the file to disk.
436    workbook.save("tutorial3.xlsx")?;
437
438    Ok(())
439}
440```
441
442Dates and times in Excel are floating point numbers that have a format applied
443to display them in the desired way. To handle dates and times with
444`rust_xlsxwriter` we create them using a [`ExcelDateTime`](crate::ExcelDateTime)
445instance and format them with an Excel number format.
446
447> **Reading ahead**:
448
449> If you enable the `chrono` feature in `rust_xlsxwriter`  you can also use
450> [`chrono::NaiveDateTime`], [`chrono::NaiveDate`] or [`chrono::NaiveTime`]
451> instances.
452
453[`chrono::NaiveDate`]:
454    https://docs.rs/chrono/latest/chrono/naive/struct.NaiveDate.html
455[`chrono::NaiveTime`]:
456    https://docs.rs/chrono/latest/chrono/naive/struct.NaiveTime.html
457[`chrono::NaiveDateTime`]:
458    https://docs.rs/chrono/latest/chrono/naive/struct.NaiveDateTime.html
459
460In the example above we create the `ExcelDateTime` instance from the date
461strings in our input data and then add a number format it so that it appears
462correctly in Excel:
463
464```text
465        let date = ExcelDateTime::parse_from_str(expense.2)?;
466        worksheet.write_with_format(row, 2, &date, &date_format)?;
467```
468
469Another addition to our program is the make the "Date" column wider for
470clarity using the
471[`Worksheet::set_column_width()`](crate::Worksheet.set_column_width) method.
472
473
474```text
475    worksheet.set_column_width(2, 15)?;
476```

568## Tutorial Part 5: Making the code more programmatic
569
570The previous example worked as expected but it it contained some hard-coded cell
571ranges like `set_values("Sheet1!$B$2:$B$5")` and `Formula::new("=SUM(B2:B5)")`.
572If our example changed to have a different number of data items then we would
573have to manually change the code to adjust for the new ranges.
574
575Fortunately, these hard-coded values are only used for the sake of a tutorial and
576`rust_xlsxwriter` provides APIs to handle these more programmatically.
577
578Let's start by looking at the chart ranges:
579
580```text
581    chart
582        .add_series()
583        .set_categories("Sheet1!$A$2:$A$5")
584        .set_values("Sheet1!$B$2:$B$5");
585```
586
587In general `rust_xlsxwriter` always provides numeric APIs for any ranges in
588Excel but when it makes ergonomic sense it also provides **secondary** string
589based APIs. The previous example uses one of these secondary string based APIs
590for demonstration purposes but for real applications you would set the chart
591ranges using 5-tuple values like this:
592
593```text
594    chart
595        .add_series()
596        .set_categories(("Sheet1", first_row, item_col, last_row, item_col))
597        .set_values(("Sheet1", first_row, cost_col, last_row, cost_col));
598```
599
600Where the range values are set or calculated in the code with something like the
601following:
602
603```text
604    let first_row = 1;
605    let last_row = first_row + (expenses.len() as u32) - 1;
606    let item_col = 0;
607    let cost_col = 1;
608```
609
610This allows the range to change dynamically if we add new elements to our `data`
611vector and ensures that the worksheet name is quoted properly (when
612required).
613
614The other section of the code that had a hard-coded string is the formula
615`"=SUM(B2:B5)"`. There isn't a single API change that can be applied to ranges
616in formulas but `rust_xlsxwriter` provides several utility functions that can
617convert numbers to string ranges. For example the
618[`cell_range()`](crate::utility::cell_range) function which takes zero indexed
619numbers and converts them to a string range like `B2:B5`:
620
621
622```text
623    let range = cell_range(first_row, cost_col, last_row, cost_col);
624    let formula = format!("=SUM({range})");
625    worksheet.write_with_format(row, 1, Formula::new(formula), &money_format)?;
626```
627
628
629> **Reading ahead**:
630>
631> The `cell_range()` function and other similar functions are detailed in the
632> [`utility`](crate::utility) documentation.
633
634Adding these improvements our application changes to the following:

682    }
683
684    // For clarity, define some variables to use in the formula and chart ranges.
685    // Row and column numbers are all zero-indexed.
686    let first_row = 1; // Skip the header row.
687    let last_row = first_row + (expenses.len() as u32) - 1;
688    let item_col = 0;
689    let cost_col = 1;
690
691    // Write a total using a formula.
692    worksheet.write_with_format(row, 0, "Total", &bold)?;
693
694    let range = cell_range(first_row, cost_col, last_row, cost_col);
695    let formula = format!("=SUM({range})");
696    worksheet.write_with_format(row, 1, Formula::new(formula), &money_format)?;
697
698    // Add a chart to display the expenses.
699    let mut chart = Chart::new_pie();
700
701    // Configure the data series for the chart.
702    chart
703        .add_series()
704        .set_categories(("Sheet1", first_row, item_col, last_row, item_col))
705        .set_values(("Sheet1", first_row, cost_col, last_row, cost_col));
706
707    // Add the chart to the worksheet.
708    worksheet.insert_chart(1, 4, &chart)?;
709
710    // Save the file to disk.
711    workbook.save("tutorial5.xlsx")?;
712
713    Ok(())
714}
