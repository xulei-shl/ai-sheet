# pi Agent 提示词构成分析

## Part 1：系统提示词（System Message）构建

### 1.1 构建流程

```
初始化阶段 ──→ agent.ts:67-88
  DefaultResourceLoader 加载 SYSTEM.md / AGENTS.md / skills / context files
  systemPromptOverride 显式注入 .pi/SYSTEM.md
  agentsFilesOverride 显式注入 .pi/AGENTS.md

重建阶段 ──→ agent-session.js:_rebuildSystemPrompt()
  每次 session 创建或 tools 变更时调用
  从 resource loader 读取所有资源 → 调用库 buildSystemPrompt() 组装

每轮调用前 ──→ agent-session.js:prompt()
  确保 this.agent.state.systemPrompt 指向最新 _baseSystemPrompt
```

### 1.2 最终 System Message 层级结构

| 顺序 | 层级 | 来源 | 加载方式 | 格式 |
|------|------|------|----------|------|
| 1 | `customPrompt` | `.pi/SYSTEM.md` | `systemPromptOverride` 显式读取 | 原始文本 |
| 2 | `appendSystemPrompt` | `.pi/APPEND_SYSTEM.md`（可选） | `DefaultResourceLoader` 自动发现 | 追加文本 |
| 3 | `<project_context>` | `.pi/AGENTS.md` + 向上遍历的 AGENTS.md | `agentsFilesOverride` 显式注入 + 自动发现 | XML `<project_instructions>` |
| 4 | `<available_skills>` | `.pi/skills/*/SKILL.md` | `DefaultResourceLoader` 自动发现 | XML skill 列表（仅 read tool 可用时） |
| 5 | `Current date` | `new Date()` | 库内生成 | 纯文本 |
| 6 | `Current working directory` | `cwd` 路径 | 库内生成 | 纯文本 |

若不提供 `SYSTEM.md`，pi 使用默认身份（"You are an expert coding assistant..."）+ 内置工具列表 + 工具指南 + Pi 文档路径，层级 4-6 仍然追加。

### 1.4 文件职责划分

| 文件 | 角色 | 内容 |
|------|------|------|
| `.pi/SYSTEM.md` | Agent 身份定义 | "你是 AI-Sheet，一个专业的 Excel 智能数据处理助手" |

### 1.6 自动上下文压缩（Compaction）

`SettingsManager.inMemory()` 默认 `compaction.enabled: true`，会在长对话中自动对早期消息做摘要压缩，减少 token 消耗。

**本项目配置**（`.pi/settings.json`）：

```json
{
  "compaction": {
    "enabled": true,
    "reserveTokens": 32768,
    "keepRecentTokens": 40000
  }
}
```

- 触发条件：`contextTokens > contextWindow - reserveTokens`
- 默认 reserveTokens = 16384，本项目设为 32768
- `keepRecentTokens` 控制保留的近期上下文量，默认 20000，本项目设为 40000
- 此配置通过 `DefaultResourceLoader` 自动发现并生效，无需修改 `agent.ts` 代码

## Part 2：用户消息（User Message）流

### 2.1 Conversation History

pi-agent 的 `SessionManager.inMemory()` 管理对话历史，每次 API 调用都携带完整历史：

```
[
  { role: "user", content: "用户第一条消息" },
  { role: "assistant", content: "AI 回复 + tool_calls" },
  { role: "user", content: "当前用户消息" }
]
```

### 2.2 两种消息注入方式

| 机制 | Role | 何时被消费 | 用途 | 代码 |
|------|------|-----------|------|------|
| `session.steer(text)` | user（排队） | 当前工具调用结束后、下一轮 LLM 调用前 | Excel 上下文、cwd 变更等动态通知 | `main.ts:243-276` |
| `session.prompt(content)` | user | 立即 | 用户请求（文本或拼装后的快捷 prompt） | `main.ts:143-241` |

### 2.3 三种输入的用户消息路径

**（1）Excel 加载 → `session.steer()`**

```
DataPage 选择 sheets/列
  → notifyContextChange()
  → steerAgent(context) → Tauri IPC → Rust stdin
  → main.ts handleSteer()
  → 组装文本：
    "[系统上下文更新] 当前文件：sales.xlsx (.../path) -> Sheet1[A(日期), B(销售额)]
     当前工作目录：C:\data
     样例数据:
     | 日期 | 销售额 |
     | --- | --- |
     | 2024-01-01 | 1000 |"
  → session.steer(contextText)
```

LLM 侧：出现在对话 history 中，作为 `{role: "user"}` 消息，排在当前消息之前。

**（2）快捷提示词 → `session.prompt()`（三段式拼接）**

```
QuickActionBar 点击「公式生成」
  → buildDirectPrompt(template, excelContext, userInput)
  → 拼接：
    "你是一个专业的 Excel 助手。

     # 当前 Excel 上下文
     文件: sales.xlsx
     Sheet: Sheet1
     列(Sheet1): A(日期), B(销售额), C(成本)

     样例数据（前 3 行）:
     | 日期 | 销售额 | 成本 |
     | --- | --- | --- |
     | 2024-01-01 | 1000 | 600 | ...

     # 用户具体需求
     计算每行的利润率

     ---
     {模板内容}"
  → sendMessage(input, displaySummary, fullPrompt)
  → sendAgentMessage(fullPrompt) → session.prompt()
```

**（3）用户文本 → `session.prompt()`**

```
AgentInput 输入 "帮我看看这个表格"
  → sendMessage(content)
  → sendAgentMessage("帮我看看这个表格")
  → session.prompt("帮我看看这个表格")
```

### 2.4 最终 API 请求结构

```
messages: [
  // System Message（每轮都携带）
  { role: "system", content: "..." },

  // Conversation History
  { role: "user", content: "[系统上下文更新] 当前文件：sales.xlsx ..." },
  { role: "assistant", content: "..." },

  // 当前 User Message
  { role: "user", content: "你是一个专业的 Excel 助手。\n\n# 当前 Excel 上下文\n..." }
  // 或
  { role: "user", content: "帮我看看这个表格" }
]
```
